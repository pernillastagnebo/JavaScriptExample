Kort om JavaScript

Under mitten av 1990-talet utvecklade företaget Netscape ett nytt programmeringsspråk som de valde att kalla för JavaScript. Netscape har även utvecklat webbläsarna Mosaic och Mozilla Firefox.

JavaScript används ofta till att skapa interaktiva webbsidor. Du kan till exempel skapa animationer, spel eller popup-rutor. Har du ett formulär kan du koda så att användaren måste fylla i ett visst antal tecken, alternativt en komplett mailadress, för att kunna gå vidare på sidan.

JavaScript-kod kan du skriva direkt i HTML dokumentet, men även i en separat fil som du sedan kopplar i din HTML-kod (rekommenderas). Du behöver inte validera din JavaScript-kod i en särskild validator utan webbsidan validerar den automatiskt. Funkar inte dina funktioner så är sidan inte heller validerad.

JavaScript www.white.se

I dagens webbsidor kan man hitta mycket JavaScript och för att visa ett exempel har jag valt att undersöka webbsidan för arkitekturfirman White närmare.

Första anblicken på webbsidan är ett stort foto som täcker hela skärmen och en meny högst upp. Efter ungefär en sekund rullas menyn upp som en rullgardin och kvar blir en liten flik med en pil på. För att få fram menyn igen får man klicka på fliken. Detta är typiskt JavaScript och kallas för dialogrutor.

Företaget vill att du ska fokusera på fotot i första hand, då det visar vad arkitektfirman har åstadkommit. Stannar du kvar på sidan ser du även att fotona ändras antingen automatiskt eller om du själv väljer att byta bild. Bildspel är också en funktion du kan åstadkomma med JavaScript.

Går man vidare in på projekt-fliken får man upp alla projekt som arkitekturfirman har åstadkommit. Man kan välja att sortera projekten från A-Ö eller efter årtal och i ikoner eller i lista. Att sortera listor på olika sätt är också en funktion som du kan göra med JavaScript.

Algoritmer

En algoritm är en steg-för-steg beskrivning av instruktioner för att lösa ett problem. Men vad har det med programmering och javascript att göra?

För att skapa ett program eller en webbsida behöver du formulera vad det är för uppgift som ska lösas, vilka steg som ska tas för att lösa uppgiften och vilka instruktioner som ska kan användas för att utföra varje delsteg (algoritm).

En algoritm skriver du i kodningen av Javascript när du ska formulera de olika stegen. Om användaren exempelvis ska gissa vilken siffra du tänker på får personen olika svar beroende på gissning. Är det får lågt får användaren ett meddelande om det, är gissningen på ett för högt tal kommer det ett meddelande om det. Gissar användaren rätt kanske det kommer upp ett ”Grattis” eller liknande. Vid mindre program kan du skriva koden direkt, men om programmet är större kan det vara bra att använda sig av pseudokod som ett enklare sätt att utrycka sig.

DOM

Förkortningen står för Document Object Model och är en av W3C:s standarder. DOM kan liknas med ett träd av dokument och dess grenar. Exempelvis är html stammen som har två grenar vid namn head och body. Underelementen i body har förmodligen ytterligare grenar. Sista elementet i ledet kallas löv. Man kan även jämföra det med familjer där löven är barn och grenarna är föräldrar och så vidare.

Ger du en förälder en egenskap kommer även denna egenskap att beröra barnet. Vill du ge ett element en egenskap så kan du använda id eller class. Du kan sätta class på flera olika element medan ett ID är unikt för ett element.

DOM är viktigt för Javascript då du ofta kopplar element med olika funktioner. Du kan exempelvis programmera så att alla p-taggar är dolda tills användaren klickar på en knapp eller rubrik. Du kan även ändra en webbsidas utseende med ett knapptryck.
